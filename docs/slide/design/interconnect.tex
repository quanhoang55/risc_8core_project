\subsubsection{Bộ phân xử và kết nối đa nhân (Interconnect)}

\begin{frame}{Arbiter Round-Robin: Chia Quyền Truy Cập Bus}
    \small
    \texttt{arbiter.v} nhận vector \texttt{request[7:0]} và chọn một core được cấp bus trong mỗi lượt cấp quyền.

    \vspace{0.15cm}
    \begin{columns}[T]
        \begin{column}{0.49\textwidth}
            \begin{block}{Cơ chế cấp quyền}
                \scriptsize
                \begin{itemize}
                    \item \texttt{grant[7:0]} là one-hot, chỉ core thắng arbitration được bật bit.
                    \item \texttt{grant\_valid} báo grant hợp lệ trong một chu kỳ.
                    \item \texttt{grant\_id} cho \texttt{bus\_ctrl} biết core nào đang được phục vụ.
                    \item \texttt{priority\_ptr} dịch sang core kế tiếp sau mỗi grant.
                    \item \texttt{last\_granted} tránh cấp lại ngay cho cùng request cũ.
                \end{itemize}
            \end{block}
        \end{column}

        \begin{column}{0.49\textwidth}
            \centering
            \scriptsize

\end{frame}

\begin{frame}{Bus Controller: Định Tuyến Giao Dịch Read/Write}
    \small
    Sau khi arbiter cấp quyền, \texttt{bus\_ctrl.v} chọn tín hiệu của core thắng và điều khiển giao tiếp với \texttt{data\_mem}.

    \vspace{0.15cm}
    \begin{center}
        \begin{tikzpicture}[
            stepbox/.style={draw=DeepBlue, fill=blue!8, minimum width=2.05cm, minimum height=0.58cm, font=\scriptsize\bfseries, align=center},
            arrow/.style={->, >=stealth, thick, DeepBlue!80}
        ]
            \node[stepbox] (s1) at (0, 0) {Core request};
            \node[stepbox] (s2) at (2.65, 0) {Arbiter grant};
            \node[stepbox] (s3) at (5.3, 0) {\texttt{bus\_ctrl}\\chọn core};
            \node[stepbox] (s4) at (7.95, 0) {\texttt{data\_mem}\\read/write};
            \node[stepbox] (s5) at (10.6, 0) {\texttt{core\_ready}};
            \draw[arrow] (s1) -- (s2);
            \draw[arrow] (s2) -- (s3);
            \draw[arrow] (s3) -- (s4);
            \draw[arrow] (s4) -- (s5);
        \end{tikzpicture}
    \end{center}

    \vspace{0.15cm}
    \begin{columns}[T]
        \begin{column}{0.48\textwidth}
            \begin{block}{Read}
                \scriptsize
                \begin{itemize}
                    \item Bật \texttt{dm\_re} và đưa \texttt{dm\_addr} sang RAM.
                    \item Nhận \texttt{dm\_rdata} từ memory interface.
                    \item Ghi dữ liệu vào \texttt{core\_rdata[grant\_id]} và bật \texttt{core\_ready}.
                \end{itemize}
            \end{block}
        \end{column}
        \begin{column}{0.48\textwidth}
            \begin{block}{Write}
                \scriptsize
                \begin{itemize}
                    \item Bật \texttt{dm\_we}, đưa \texttt{dm\_addr} và \texttt{dm\_wdata}.
                    \item \texttt{data\_mem} ghi dữ liệu theo cạnh clock.
                    \item \texttt{bus\_ctrl} báo \texttt{core\_ready} để core kết thúc giao dịch.
                \end{itemize}
            \end{block}
        \end{column}
    \end{columns}

    \vspace{0.05cm}
    \centering
    \scriptsize
    Trong thiết kế hiện tại, read ở memory interface là combinational/async; write là synchronous trong \texttt{data\_mem}.
\end{frame}

\begin{frame}{Atomic LR/SC Và AMO Trong Interconnect}
    \small
    Interconnect có primitive phần cứng cho đồng bộ đa lõi thông qua reservation và phép toán nguyên tử trên RAM dùng chung.

    \vspace{0.15cm}
    \begin{columns}[T]
        \begin{column}{0.49\textwidth}
            \begin{block}{LR/SC Reservation}
                \scriptsize
                \begin{itemize}
                    \item \texttt{LR.W}: đọc memory và đặt \texttt{reservation\_valid[core]} tại \texttt{reservation\_addr[core]}.
                    \item \texttt{SC.W} thành công nếu reservation còn đúng địa chỉ; khi đó RAM được ghi và kết quả trả về là thành công.
                    \item Nếu core khác ghi cùng địa chỉ, reservation bị invalidate; \texttt{SC.W} sau đó thất bại.
                \end{itemize}
            \end{block}
        \end{column}

        \begin{column}{0.49\textwidth}
            \begin{alertblock}{AMO}
                \scriptsize
                \begin{itemize}
                    \item AMO đọc giá trị cũ từ RAM.
                    \item \texttt{bus\_ctrl} tính giá trị mới theo \texttt{amo\_op}: add, swap, and, or, xor, min/max.
                    \item RAM nhận giá trị mới; core nhận lại old value qua \texttt{core\_rdata}.
                \end{itemize}
            \end{alertblock}
        \end{column}
    \end{columns}

    \vspace{0.15cm}
    \begin{center}
        \scriptsize
        Đây là nền tảng phần cứng để xây dựng cơ chế đồng bộ; slide này không trình bày như một mutex/semaphore phần mềm hoàn chỉnh.
    \end{center}
\end{frame}
