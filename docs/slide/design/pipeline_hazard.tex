\subsubsection{Bộ xử lý xung đột (Hazard Unit)}
\begin{frame}{Quản lý xung đột trong Pipeline (\texttt{hazard\_unit})}
    Quá trình chạy gối đầu lệnh (Pipelining) sinh ra các xung đột dữ liệu và điều khiển, được xử lý trực tiếp bên trong \texttt{risc\_core.v}:
    \vspace{0.2cm}
    
    \begin{columns}
        \begin{column}{0.5\textwidth}
            \begin{block}{1. Xung đột dữ liệu (Data Hazard)}
                Xảy ra khi lệnh sau cần đọc giá trị thanh ghi mà lệnh trước chưa ghi xong.
                \begin{itemize}
                    \item \textbf{Mạch Forwarding (Bơm tắt):} Chuyển kết quả trực tiếp từ ngõ ra \texttt{EX/MEM} hoặc \texttt{MEM/WB} về ngõ vào ALU mà không cần chờ Write Back.
                    \item \textbf{Load-use Stall:} Với lệnh Load, kích hoạt tín hiệu \texttt{load\_use\_stall} để hoãn Pipeline 1 chu kỳ khi lệnh kề sau sử dụng kết quả Load.
                \end{itemize}
            \end{block}
        \end{column}
        
        \begin{column}{0.48\textwidth}
            \begin{alertblock}{2. Xung đột điều khiển (Control)}
                Xảy ra khi các lệnh nhảy hoặc rẽ nhánh thay đổi dòng chảy địa chỉ \texttt{PC}.
                \begin{itemize}
                    \item Khi tín hiệu \texttt{redirect\_taken} kích hoạt (do Branch hoặc Jump).
                    \item \textbf{Cơ chế Flush (Xóa):} Reset các thanh ghi đệm tầng nạp sai lệnh (\texttt{if\_id\_valid <= 0}, \texttt{id\_ex\_valid <= 0}), đưa các tín hiệu điều khiển về 0 (tạo lệnh rỗng \texttt{NOP}).
                \end{itemize}
            \end{alertblock}
        \end{column}
    \end{columns}
\end{frame}
